## Documentation Standards

All documentation in this directory follows these standards:
- **Markdown format** for easy reading and version control
- **Clear headings** and table of contents for navigation
- **Code examples** with syntax highlighting
- **Screenshots** where applicable (in `images/` subdirectory)
- **Cross-references** between related documents
- **Version information** and last updated dates

## Contributing to Documentation

When updating documentation:
1. Keep the main [README.md](../README.md) as the primary entry point
2. Update relevant documents in this directory for detailed information
3. Maintain cross-references between documents
4. Include practical examples and use cases
5. Test all code examples and commands

## Getting Started

If you're new to the System Monitor project:
1. Start with the main [README.md](../README.md) for a quick overview
2. Read the [Architecture Overview](architecture.md) to understand the system
3. Follow the [User Manual](user-manual.md) to use the dashboard
4. Check the [Developer Guide](developer-guide.md) if you want to contribute
